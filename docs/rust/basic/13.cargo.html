1 配置文件

发布的配置信息, 是预定义好的,可自定义

cargo build
    Finished dev [unoptimized + debuginfo] target(s) in 0.0s
cargo build --release
    Finished release [optimized] target(s) in 0.0s

我们运行2个命令看看, 看到 dev 和 release, 实际就是使用了不同的默认配置 dev默认的一些配置

[profile.dev]
opt-level = 0
debug = true
split-debuginfo = '...'  # Platform-specific.
strip = "none"
debug-assertions = true
overflow-checks = true
lto = false
panic = 'unwind'
incremental = true
codegen-units = 256
rpath = false

我们可以通过修改Cargo.toml来覆盖一些默认配置

[package]
name = "world"
version = "0.1.0"
edition = "2021"
[dependencies]
rand = "0.8.5"
[profile.dev] # dev 配置
opt-level = 1
[profile.release] # release 配置
opt-level = 3

修改完后, 我们再次编译可以看到 optimized

cargo build
    Finished dev [optimized + debuginfo] target(s) in 3.14s

dependencies的三种方式

[dependencies]
clap = "2.27.1" # from crates.io
rand = { git = "https://github.com/rust-lang-nursery/rand" } # from online repo
bar = { path = "../bar" } # from a path in the local filesystem

2 文档

2.1 注释

生成HTML文档,显示如何使用API

我们 在src/lib.rs中添加一些文档注释

src/lib.rs
1//! # My Crate
//!
//! `my_crate` is a collection of utilities to make performing certain
//! calculations more convenient.

/// Adds one to the number given.
///
2/// # Examples
///
/// ```
3/// let arg = 5;
/// let answer = my_crate::add_one(arg);
///
/// assert_eq!(6, answer);
/// ```
pub fn add_one(x: i32) -> i32 {
    x + 1
}
1
描述 我们要发布的 整个crate, 使用 //!
2
使用 ///,可以用markdown 语法
Example 是一个常见的我们要去写的区域,后面写上示例
Panics: 描述可能发生panic的场景
Errors: 如何函数返回Result,描述可能的错误种类,以及导致错误的条件
Safty: 如果函数处于unsafe调用, 那么就描述使用unsafe的原因和使用的前提
3
使用cargo test 会测试这里的代码, 注意 需要将 示例代码里使用的 my_crate改成 Cargo.toml中 [package] 的 name 
# --open 会直接打开 生成的 html
cargo doc --open
# 会测试 文档注释中的代码
cargo test
   Doc-tests ....

2.2 pub use

  • src/lib.rs
  • src/main.rs
//! # Art
//!
//! A library for modeling artistic concepts.

// --加上这些--
1pub use self::kinds::PrimaryColor;
pub use self::kinds::SecondaryColor;
pub use self::utils::mix;
//  -----

pub mod kinds {
    /// The primary colors according to the RYB color model.
    pub enum PrimaryColor {
        Red,
        Yellow,
        Blue,
    }

    /// The secondary colors according to the RYB color model.
    pub enum SecondaryColor {
        Orange,
        Green,
        Purple,
    }
}

pub mod utils {
    use crate::kinds::*;

    /// Combines two primary colors in equal amounts to create
    /// a secondary color.
    pub fn mix(c1: PrimaryColor, c2: PrimaryColor) -> SecondaryColor {
        SecondaryColor::Orange
    }
}
1
这样生成的文档方便用户查看,不需要点击多次才能看到 
// use art::kinds::PrimaryColor;
// use art::utils::mix;

// 在lib.rs 添加 pub use 后, 这里可以这样了,生成的文档也方便用户使用
use art::mix;
use art::PrimaryColor;

fn main() {
    let red = PrimaryColor::Red;
    let yellow = PrimaryColor::Yellow;
    mix(red, yellow);
}

3 发布crate到crate.io

3.1 登录crate.io获取token

登录crate.io后,进入settings页面, 生成一个token

# 该命令会将token 存储在 ~/.cargo/credentials
cargo login abcdefghijklmnopqrstuvwxyz012345

3.2 修改Cargo.toml

[package]
1name = "guessing_game"
version = "0.1.0"
edition = "2021"
2description = "该描述会显示在crate.io的搜索结果里"
3license = "MIT OR Apache-2.0"
[dependencies]
1
名字在crate.io中必须是独一无二的
2
必填项
3
必填项,许可标识值可在spdx上查找

3.3 发布

cargo publish

发布已存在crate的新版本, 需要修改 toml文件中的 version 参考 http://semver.org/, 再执行cargo publish 即可

3.4 cargo yank 撤回

  1. 首先 crate.io 是不能让你删除已经发布的版本的,因为如果已经有人用了怎么办
  2. 但是可以让你撤回,它的意思是, 防止新的用户使用这个可能不太好的版本,cargo add xxxx 后不会使用到这个撤回的版本, 如果是已经使用了这个版本的人(Cargo.lock里指定了版本了), 还是可以下载使用的
cargo yank --vers 1.0.1
    Updating crates.io index
        Yank guessing_game@1.0.1
# 取消 yank
cargo yank --vers 1.0.1 --undo
    Updating crates.io index
      Unyank guessing_game@1.0.1

4 工作空间

4.1 创建

mkdir add
cd add 
touch Cargo.toml

resolver-versions

Cargo.toml
[workspace]

members = [
    "adder",
]
resolver = "2"
# 添加 binary crate
cargo new adder
cargo build
tree -L 2
    .
    ├── Cargo.lock
    ├── Cargo.toml
    ├── adder
    │   ├── Cargo.toml
    │   └── src
    └── target # 存放所有members 编译后的东西
        ├── CACHEDIR.TAG
        └── debug
cd adder
# 我们没有看到在这个adder目录下生成target目录
# 还是会在 外层的 target 目录下生成
cargo build

# 添加一个 lib crate
# 该命令会在 Cargo.toml的 members 里自动添加
cargo new add_one --lib
tree -L 2
    ├── Cargo.lock
    ├── Cargo.toml
    ├── add_one
    │   ├── Cargo.toml
    │   └── src
    │       └── lib.rs
    ├── adder
    │   ├── Cargo.toml
    │   └── src
    │       └── main.rs
    └── target
在add_one/src/lib.rs添加一个方法
pub fn add_one(x: i32) -> i32 {
    x + 1
}

现在我们要在adder 中依赖add_one这个库,需要在adder/Cargo.toml中添加

adder/Cargo.toml
[package]
name = "adder"
version = "0.1.0"
edition = "2021"

[dependencies]
add_one = { path = "../add_one" }
adder/src/main.rs
use add_one;

fn main() {
    let num = 10;
    println!("Hello, world! {num} plus one is {}!", add_one::add_one(num));
}
# 运行指定 binary crate
cargo run  -p adder

4.2 使用外部crate

adder/Cargo.toml[dependencies]中添加rand = "0.8.1",然后在add_one/Cargo.toml[dependencies]中添加rand = "0.8.5"后, cargo build 只会在外层Cargo.lock中生成一个rand版本.这样工作空间中的crate 都会使用同一个版本的rand

6 自定义命令扩展

在系统中你可以直接运行的一个命令 名字是cargo-xxxx,那么你就可以运行cargo xxxx 这样来替代运行它, cargo --list 可以查看到.

我们在安装rust后, 可以在~/.cargo/bin 看到一些可执行文件, cargo-fmt 就是其中的一个, 我们可以发现有cargo fmt这个命令,就像cargo 的内置命令一样
