1 闭包

1.1 定义

Tip
  • closures can capture values from the scope in which they’re defined
  • 可以捕获其所在环境的匿名函数
    • 在它定义的作用域中捕获值
fn main() {
    fn add_one_v1(x: u32) -> u32 {
        x + 1
    }
    // 闭包,匿名函数, |参数|->返回值 { 函数体}
    let add_one_v2 = |x: u32| -> u32 { x + 1 };
    // 等价与 |x| {x + 1}  因为只有一个表达式, 所以 {} 可以省略
    let add_one_v3 = |x| x + 1;
    // 上面的定义没有类型, 如果没有的调用语句, 是会报错的, 因为函数没有类型,内存占用无法确定
    // 调用后, 编译器推断了类型,这个时候类型就确定了
    let r = add_one_v3(1);
    println!("{}", r);
    // 后续如果你用其他类型,就会报错的
    // let r2 = add_one_v3("a");
}

1.2 捕获值的方式

不可变借用

fn main() {
    let list = vec![1, 2, 3];
    println!("Before defining closure: {:?}", list);
    // 捕获了闭包所在环境, 变量list
    let only_borrows = || {
        // 这里的list,就是外面的那个list,不是赋值操作
        // 只有我们指定 move|| {} 时才会 转移所有权
        // println! 本身使用list的不可变借用
        println!("From closure: {:?}", list);
    }

    println!("Before calling closure: {:?}", list);
    only_borrows();
    println!("After calling closure: {:?}", list);
}

可变借用

fn main() {
    let mut list = vec![1, 2, 3];
    println!("Before defining closure: {:?}", list);
    // 需要mut
    let mut borrows_mutably = || list.push(7);

    borrows_mutably();
    println!("After calling closure: {:?}", list); // [1, 2, 3, 7]
}

move 所有权

use std::thread;

fn main() {
    let list = vec![1, 2, 3];
    println!("Before defining closure: {:?}", list);
    // 指定move, 所有权移动
    thread::spawn(move || println!("From thread: {:?}", list))
        .join()
        .unwrap();
    // println!("after defining closure: {:?}", list); // 被move了,报错
}

1.3 fn trait

Tip
  • 闭包可能实现了Fn,FnMut,FnOnce这三个trait中的一个或多个
  • 创建闭包时,通过闭包对环境值的使用情况,rust推断出闭包实现了哪些trait

1.3.1 FnOnce

FnOnce表示的意思
  1. 可以被调用一次,很显然我们编写的闭包肯定是要能被调用至少一次的.
  2. 如果只实现了 FnOnce这个trait, 其他trait没有实现,那么表示该闭包只能被调用一次
  3. 当你闭包捕获的环境值离开闭包它自己的body时(值无效了),它就是 只实现了 FnOnce,因为它被调用一次后,里面变量值无效了,再调用就报错了
fn main() {
    let farewell = "goodbye".to_owned();
    let diary = move || {
        println!("I said {}.", farewell);
    };
    diary(); // ok
    diary(); // ok
}
// trait 是FnOnce , 表示传递的参数必须是实现了FnOnce的闭包
// 这里的写法 和一般的trait 不太一样
fn apply<F>(f: F)
where
    F: FnOnce(),
{
    f();
}
fn main() {
    let mut farewell = "goodbye".to_owned();
1    let diary = move || {
        println!("I said {}.", farewell);
        {
            farewell;
        } // farewell 被drop掉了,
    };
    apply(diary);
    // diary();
    // 我们可以推断, 因为闭包内部使用的farewell 已经drop掉了,再次调用就有问题了
    // 闭包对应的汇编指令,使用的farewell的地址是确定的
    // 所以报错了, 提示diary 变量本身被move掉了
    // 你可以认为因为你这样使用捕获的变量,导致 该闭包没有实现 copy,使用时就是move操作
    diary();
    // 可以用这个,也是报错了
    // println!("{:p}", &diary);
}
1
不用move 去捕获变量,这里的结果还是一样的,主要是看你闭包内部是如何使用这个捕获的变量

看这个例子,只能调用一次

fn main() {
    let mut farewell = "goodbye".to_owned();
    let diary = || {
        println!("I said {}.", farewell);
        // 将farewell 作为返回值move给了 diary
        // 我们没有使用move, 这个farewell 就是外面的那个变量本身
        // 如果使用move || {}, 只不过将外部的move给这里的farewell
        farewell

}

fn main() {
    let farewell = "goodbye".to_owned();
    let diary = move || {
        println!("I said {}.", farewell);
    };
    diary();
    diary();
}

1.3.2 FnMut

Tip
  1. 闭包可以修改它捕获的变量,但是不能move 捕获的值 out of their body
  2. 说到底就是能够调用多次(内部使用的变量多次调用还有效),还能修改捕获的变量
fn apply<F>(mut f: F)
where
    F: FnMut(),
{
    f();
    f();
}
fn main() {
    let mut farewell = "goodbye".to_owned();
    // 使用 move || {} 结果也是一样的
    // 还是那句话, 关键在闭包内部如何使用这个捕获的值
    let mut diary = || {
        farewell.push_str("hello");
        // 内部修改捕获的值, 值没有失效
        println!("I said {}.", farewell);
    };
    diary(); // 调用本身需要 let mut diary
    apply(diary);
}

看看这个…

fn apply<F>(mut f: F)
where
    F: FnMut(),
{
    f();
}
fn main() {
    let x = "goodbye".to_owned();
    let print = || {
        println!("{}", x);
    };
    apply(print);
}

1.3.3 Fn

Tip

不修改捕获的值,不move 捕获的值 out of their body,或者那种不捕获任何值的闭包

fn apply<F>(f: F)
where
    F: Fn(),
{
    f();
}
// 也是实现了 Fn() trait
fn function() {
    println!("I'm a function!");
}
fn main() {
    let x = 7;
    let print = || {
        println!("{}", x);
    };
    apply(print);
    apply(function);
}

1.4 闭包作为返回值

fn create_fn() -> impl Fn() {
    let text = "Fn".to_owned();
    // move 必须要有,要不然最后 text 会被 drop
    move || println!("This is a: {}", text)
}

fn create_fnmut() -> impl FnMut() {
    let text = "FnMut".to_owned();
    // 同样move
    move || println!("This is a: {}", text)
}

fn create_fnonce() -> impl FnOnce() {
    let text = "FnOnce".to_owned();

    move || println!("This is a: {}", text)
}

fn main() {
    let fn_plain = create_fn();
    let mut fn_mut = create_fnmut();
    let fn_once = create_fnonce();

    fn_plain();
    fn_mut();
    fn_once();
}

2 iterator 迭代器

Tip

除非调用迭代器的方法,否则迭代器本身没有任何消耗

2.1 简单使用

let v1 = vec![1, 2, 3];
// 仅仅定义而没有使用它, 这样不会有任何的消耗
let v1_iter = v1.iter();
fn main() {
    let v1 = vec![1, 2, 3];

    // v1_iter 本身取得的是 v1元素的不可变借用
    let v1_iter = v1.iter();

    // for循环 会取得v1_iter的所有权
    // 底层是调用next
    for val in v1_iter {
        println!("Got: {}", val);
    }
    //  v1_iter 已经被move了, 报错
    // println!("{:?}", v1_iter);

    let mut v1_iter = v1.iter();
    if let Some(b) = v1_iter.next() {
        println!("{}", b);
    }
    // 调用next 会消耗迭代器 (体会一下消耗的意思)
    v1_iter.next();
    v1_iter.next();
    if let None = v1_iter.next() {
        println!("没有元素后再调用next 返回None");
    }

    let v1_iter = v1.iter();
    let total: i32 = v1_iter.sum();
    println!("{total}");
    // 调用sum 会获取v1_iter的所有权,所以报错了,提示move掉了
    // println!("{:?}", v1_iter);

    // into_iter 会获取v1的所有权
    let v1_iter = v1.into_iter();
    // println!("{:?}", v1); 报错

    let mut v1 = vec![1, 2, 3];
    // 取得v1的可变借用
    let v1_iter = v1.iter_mut();

}

2.2 迭代器适配器

Tip
  • 非消耗型迭代器适配器 是定义在 迭代器 trait上的一些方法,它不会消耗迭代器,相反它是产生一个新的迭代器
  • 消耗型迭代器适配器, 前面我们使用的next 就是
fn main() {
    let v1: Vec<i32> = vec![1, 2, 3];

    let v1_iter = v1.iter();
    // 取得迭代器的所有权,生成新的迭代器
    let v_map = v1_iter.map(|x| x + 1);
    // 使用消耗迭代器适配器
    let v2: Vec<_> = v_map.collect();

    assert_eq!(v2, vec![2, 3, 4]);

    let a = [1, 2, 3];
    let b = ['a', 'b', 'c'];
    println!("{:?}", a.iter().zip(b.iter()).collect::<Vec<_>>());
}

使用闭包捕获环境

fn main() {
    let v1: Vec<i32> = vec![1, 2, 3];
    // any 如果有一个满足条件就返回true
    // v1.iter() => iter(&v1)
    println!("2 in v1: {}", v1.iter().any(|&x| x == 2));

    let mut iter = v1.iter();
    println!("Find 2 in v1: {:?}", iter.find(|&&x| x == 2));

    let v1: Vec<i32> = vec![1, 2, 3];
    println!("Find 2 in v1: {:?}", v1.into_iter().find(|&x| x == 2));

    let v1: Vec<i32> = vec![1, 2, 3];
    let index_of_first_even_number = v1.iter().position(|&x| x % 2 == 0);
    assert_eq!(index_of_first_even_number, Some(5));

    let v1: Vec<i32> = vec![1, 2, 3];
    let v1_iter = v1.into_iter();
    let two = 2;
    // filter 将符合条件的返回 (true的情况)
    let v_map = v1_iter.filter(|x| x >= &two);
    let v2: Vec<_> = v_map.collect();
    println!("{:?}", v2);
}

2.3 iterator trait

所有迭代器都实现了 iterator trait

Iterator trait 在标准库中的定义
pub trait Iterator {
    type Item;

    fn next(&mut self) -> Option<Self::Item>;

    // methods with default implementations elided
}

我们自定义迭代器时,只需要实现next方法即可

// 定义一个能生成范围内所有奇数的迭代器
struct OddNumbers {
    current: u32,
    end: u32,
}

// 为 OddNumbers 实现迭代器 trait
impl Iterator for OddNumbers {
    type Item = u32; // 迭代器的项类型是 u32

    // next() 方法返回下一个奇数，如果没有更多奇数则返回 None
    fn next(&mut self) -> Option<Self::Item> {
        if self.current >= self.end {
            None // 如果当前值已经大于或等于结束值，返回 None
        } else {
            let result = self.current;
            self.current += 2; // 移动到下一个奇数
            Some(result) // 返回当前的奇数
        }
    }
}

fn main() {
    // 创建迭代器实例，生成从 1 到 9 的所有奇数
    let mut odd_numbers = OddNumbers {
        current: 1,
        end: 10,
    };

    // 通过循环来迭代奇数
    loop {
        match odd_numbers.next() {
            Some(n) => println!("{}", n),
            None => break, // 如果没有更多元素，则退出循环
        }
    }

    let odd_numbers = OddNumbers {
        current: 1,
        end: 10,
    };
    // 使用 for 循环来简化迭代
    for odd in odd_numbers {
        println!("{}", odd);
    }
}
